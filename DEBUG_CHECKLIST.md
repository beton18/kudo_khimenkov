# Чек-лист диагностики картинок

## 1. Проверь структуру репозитория на GitHub

Открой: https://github.com/beton18/kudo_khimenkov

**Должна быть папка `public/assets/` с файлами:**
- devuska-kikbokser.jpg
- bokser-v-cernom-remeske-na-zapast-e.jpg
- vid-sboku-rebenok-zanimaetsa-boksom.jpg
- zenskii-bokser-treniruetsa-s-trenerom.jpg
- mock.jpg

**Скинь скрин:** есть ли эта папка и файлы в репозитории?

---

## 2. Проверь консоль браузера

Открой сайт: https://beton18.github.io/kudo_khimenkov/

Нажми **F12** → вкладка **Console**

**Скинь:**
- Все ошибки красным цветом (особенно 404 Not Found)
- Полные пути к картинкам из ошибок

Пример ошибки:
```
GET https://beton18.github.io/assets/mock.jpg 404 (Not Found)
```

---

## 3. Проверь Network (загрузку файлов)

В DevTools (F12) → вкладка **Network**

1. Обнови страницу (F5)
2. Найди файлы с расширением .jpg
3. Кликни на каждый

**Скинь:**
- Request URL (полный путь к картинке)
- Status Code (200, 404, 403?)
- Скрин списка запросов

---

## 4. Проверь HTML код на странице

На сайте:
1. Правой кнопкой → **Inspect** (Проверить)
2. Найди элемент `<img>` с картинкой
3. Посмотри атрибут `src`

**Скинь:**
- Что в атрибуте `src=` у картинок?
- Пример: `<img src="/kudo_khimenkov/assets/mock.jpg">`

---

## 5. Попробуй открыть картинку напрямую

Вставь в адресную строку:
```
https://beton18.github.io/kudo_khimenkov/assets/mock.jpg
```

**Скинь:**
- Открылась картинка или ошибка 404?
- Если ошибка - скрин страницы ошибки

---

## 6. Проверь Actions - логи деплоя

Открой: https://github.com/beton18/kudo_khimenkov/actions

1. Кликни на последний успешный деплой (зеленая галочка)
2. Открой job "build" → шаг "Build"

**Скинь:**
- Есть ли в логах строки про копирование assets?
- Размер итогового dist

---

## 7. Проверь локально

В папке проекта запусти:
```bash
npm run build
```

Потом открой папку `dist/` и проверь:
- Есть ли там папка `assets/`?
- Есть ли в ней все 5 картинок?

**Скинь:** скрин структуры папки dist/
